Normalización





Las 5 normas o principios fundamentales que rigen el diseño y la gestión de Bases de Datos son:



  • Integridad de los Datos:

    Los datos deben ser precisos y consistentes. Se deben implementar restricciones como claves primarias,
    claves foráneas y reglas de validación para asegurar la integridad.

  • Evitar Redundancia:

    Se debe minimizar la duplicación de datos para evitar inconsistencias y ahorrar espacio. La normalización es
    una práctica común para lograr esto.

  • Consistencia:

    Los datos deben mantener un estado consistente en todo momento. Las transacciones deben seguir el principio ACID
    (Atomicidad, Consistencia, Aislamiento y Durabilidad).

  • Seguridad:

    Los datos deben estar protegidos contra accesos no autorizados. Se deben implementar controles de acceso y cifrado
    según sea necesario.

  • Escalabilidad y Rendimiento:

    La base de datos debe ser capaz de crecer y manejar un aumento en la carga de trabajo sin
    comprometer el rendimiento. Esto incluye el uso adecuado de índices y optimización de consultas.
